My Role

At JPR Systems Pvt. Ltd., I played a key role in the ongoing development of HealWeb HIS, a robust Hospital Information System used by clinics and hospitals to manage operations like HR, Pharmacy, IPD, Billing, and more.

My primary responsibilities involved continuously building new features and taking ownership of critical modules such as HR, Pharmacy, and IPD. I worked across both frontend and backend layers of the application, contributing to a wide range of high-impact tasks.

  • Developed and optimized two business-critical modules by implementing full-stack logic using Angular and ASP.NET Core.
  • Built dynamic billing and invoice generation features, and created PDF reports for various patient and billing data.
  • Improved application performance by migrating complex frontend logic to backend services, resulting in a 30% performance boost and 40% code reduction in the HR module.
  • Identified and resolved a significant memory leak in RxJS streams, enhancing the app's stability and efficiency.
  • Utilized RxJS for handling complex asynchronous flows, leading to a 15% reduction in codebase while improving functionality by 20%.
  • Optimized and refactored legacy code for better performance and maintainability across modules.

I was frequently entrusted with business-critical tasks and tight deadlines, highlighting the trust placed in my technical judgment and delivery capacity. My role often involved balancing feature delivery with maintaining system performance and code quality.

In addition to engineering work, I also contributed to team collaboration and leadership:

  • Led and mentored a team of 8–9 developers, improving productivity by 20% through better task planning, regular code reviews, and clear communication.
  • Collaborated with UI/UX designers to ensure user-friendly and intuitive interface experiences.
  • Worked closely with backend developers to ensure seamless API integration and consistent data flow throughout the application.

Appointo, Appointment Booking System

This project was a personal initiative to create a Minimum Viable Product (MVP) for a modern Appointment Booking System, designed specifically for service-based professionals. My goal was to blend smart scheduling, intuitive role-based access, and performance-first development into a flexible, real-world solution.

My Role

I built the entire system from scratch — from UI to backend logic — handling:

  • System architecture
  • Frontend & backend development
  • Role-based authentication flows
  • Database design
  • UX planning and admin dashboard layout

I approached it not just as a coder, but as a product problem-solver building something future-ready.

Key Features

  • Role-Based Access (Auth0): Separate views and permissions for Admins and Customers.
  • Smart Scheduling System: Admins can define working hours and take leaves; affected bookings are auto-rescheduled for continuity.
  • Admin Dashboard: Easily track bookings, user activity, and manage appointments.
  • Email Notifications: Triggered on booking, cancellation, and rescheduling actions.
